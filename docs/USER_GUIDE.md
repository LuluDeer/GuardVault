## 系统概述

GuardVault 是一个企业级双因素认证（2FA/TOTP）集中管理系统，主要功能包括：

- **集中管理**: 管理员统一管理所有 TOTP 服务账号
- **权限控制**: 按部门、按用户精细授权
- **多端支持**: PC 客户端、Web 管理后台
- **安全审计**: 完整的操作日志和审计报表
- **多种录入**: 手动录入、扫码、批量导入、谷歌 OTP 迁移

## 角色权限

### 1. 超级管理员 (super_admin)

**权限范围**: 全局管理权限

- 用户管理（增删改查、禁用/启用）
- 部门管理（增删改查）
- 服务管理（增删改查、批量导入）
- 授权管理（为用户授权/撤销服务）
- 审计报表查看
- 系统配置管理
- 查看所有部门的 TOTP 密钥明文

### 2. 部门管理员 (dept_admin)

**权限范围**: 仅限本部门

- 本部门用户管理
- 本部门服务管理
- 本部门授权管理
- 本部门审计报表查看
- 查看本部门的 TOTP 密钥明文

### 3. 普通用户 (user)

**权限范围**: 仅限个人

- 查看已授权的服务列表
- 获取 TOTP 验证码
- 复制验证码
- 修改个人密码
- 开启/关闭个人 2FA

## 管理员操作指南

### 1. 首次登录

#### 步骤 1: 初始化系统

首次部署后，需要初始化超级管理员账号：

```bash
# 使用 curl 初始化
curl -X POST http://your-server:3001/api/admin/init \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "Admin@123"}'

# 或在浏览器访问
# http://your-server:3001/api/admin/init
```

#### 步骤 2: 登录 Web 管理后台

1. 打开浏览器，访问 `http://your-server:3001` 或配置的域名
2. 输入用户名: `admin`
3. 输入密码: `Admin@123`
4. 点击"登录"按钮

#### 步骤 3: 修改默认密码

首次登录后，强烈建议修改默认密码：

1. 点击右上角用户名
2. 选择"修改密码"
3. 输入当前密码和新密码
4. 点击"确认"

### 2. 部门管理

#### 创建部门

1. 登录后，进入"部门管理"页面
2. 点击"新增部门"按钮
3. 填写部门信息：
   - **部门名称**: 如"IT 部门"、"运维部"
   - **部门编码**: 如"it"、"ops"（必须唯一）
   - **描述**: 部门说明（可选）
4. 点击"确认"

#### 编辑部门

1. 在部门列表中找到要编辑的部门
2. 点击"编辑"按钮
3. 修改部门信息
4. 点击"确认"

#### 删除部门

**注意**: 删除部门前，请确保：
- 部门下没有用户
- 部门下没有服务

1. 在部门列表中找到要删除的部门
2. 点击"删除"按钮
3. 确认删除操作

### 3. 用户管理

#### 创建用户

1. 进入"用户管理"页面
2. 点击"新增用户"按钮
3. 填写用户信息：
   - **用户名**: 登录账号（必须唯一）
   - **邮箱**: 邮箱地址（可选，用于找回密码）
   - **密码**: 初始密码
   - **角色**: 选择用户角色
   - **所属部门**: 选择部门
   - **状态**: 启用/禁用
4. 点击"确认"

#### 编辑用户

1. 在用户列表中找到要编辑的用户
2. 点击"编辑"按钮
3. 修改用户信息
4. 点击"确认"

#### 重置用户密码

1. 在用户列表中找到目标用户
2. 点击"重置密码"按钮
3. 输入新密码
4. 点击"确认"

#### 禁用/启用用户

1. 在用户列表中找到目标用户
2. 点击"禁用"或"启用"按钮
3. 确认操作

**注意**: 禁用用户后，该用户将无法登录系统。

#### 为用户启用 2FA

1. 在用户列表中找到目标用户
2. 点击"启用 2FA"按钮
3. 系统会显示二维码
4. 用户使用手机验证码应用扫描二维码
5. 输入验证码确认绑定

### 4. 服务管理

#### 服务分类

系统支持以下服务分类：

- **云服务**: AWS、Azure、阿里云、腾讯云等
- **VPN**: OpenVPN、WireGuard 等
- **代码托管**: GitHub、GitLab、Gitee 等
- **CI/CD**: Jenkins、GitLab CI 等
- **监控**: Prometheus、Grafana 等
- **其他**: 其他自定义服务

#### 手动录入服务

1. 进入"服务管理"页面
2. 点击"新增服务"按钮
3. 填写服务信息：
   - **服务名称**: 如"AWS 生产环境"
   - **服务分类**: 选择分类
   - **所属部门**: 选择部门
   - **账号**: 服务账号（如 root、admin）
   - **TOTP 密钥**: 输入或生成密钥
   - **算法**: 选择算法（SHA1/SHA256/SHA512）
   - **位数**: 选择位数（6/8）
   - **周期**: 选择周期（30/60 秒）
   - **描述**: 服务说明（可选）
4. 点击"确认"

#### 扫码录入服务

1. 进入"服务管理"页面
2. 点击"扫码录入"按钮
3. 使用手机扫描服务提供的二维码
4. 系统自动解析并填充信息
5. 确认并保存

#### 批量导入服务

**CSV 格式导入:**

1. 准备 CSV 文件，格式如下：

```csv
服务名称,分类,部门,账号,密钥,算法,位数,周期,描述
AWS生产环境,云服务,it,admin,JBSWY3DPEHPK3PXP,SHA1,6,30,AWS账号
GitHub账号,代码托管,it,developer,KRUGS4ZANFZSAYJA,SHA1,6,30,GitHub
```

2. 进入"服务管理"页面
3. 点击"批量导入" → "CSV 导入"
4. 上传 CSV 文件
5. 预览导入数据
6. 确认导入

**JSON 格式导入:**

1. 准备 JSON 文件，格式如下：

```json
[
  {
    "name": "AWS生产环境",
    "category": "云服务",
    "deptId": 1,
    "account": "admin",
    "secret": "JBSWY3DPEHPK3PXP",
    "algorithm": "SHA1",
    "digits": 6,
    "period": 30,
    "description": "AWS账号"
  }
]
```

2. 进入"服务管理"页面
3. 点击"批量导入" → "JSON 导入"
4. 上传 JSON 文件
5. 预览导入数据
6. 确认导入

#### 谷歌验证码迁移导入

**方式一：粘贴迁移 URL**

1. 在谷歌验证码应用中选择"转移账号"
2. 扫描显示的二维码获取迁移 URL（以 `otpauth-migration://` 开头）
3. 进入"服务管理"页面
4. 点击"批量导入" → "谷歌 OTP 导入"
5. 选择"粘贴迁移 URL"
6. 粘贴迁移 URL
7. 点击"解析"
8. 选择要导入的服务
9. 修改分类和部门
10. 确认导入

**方式二：上传二维码图片**

1. 在谷歌验证码应用中选择"转移账号"
2. 截图保存迁移二维码
3. 进入"服务管理"页面
4. 点击"批量导入" → "谷歌 OTP 导入"
5. 选择"上传二维码图片"
6. 上传二维码图片
7. 系统自动解析
8. 选择要导入的服务
9. 修改分类和部门
10. 确认导入

**方式三：上传导出文件**

1. 将迁移 URL 保存为 .txt 文件
2. 进入"服务管理"页面
3. 点击"批量导入" → "谷歌 OTP 导入"
4. 选择"上传导出文件"
5. 上传 .txt 文件
6. 系统自动解析
7. 选择要导入的服务
8. 修改分类和部门
9. 确认导入

#### 编辑服务

1. 在服务列表中找到要编辑的服务
2. 点击"编辑"按钮
3. 修改服务信息
4. 点击"确认"

#### 查看服务密钥明文

1. 在服务列表中找到目标服务
2. 点击"查看密钥"按钮
3. 系统会显示密钥明文和二维码
4. **注意**: 此操作会记录审计日志

#### 重置服务密钥

1. 在服务列表中找到目标服务
2. 点击"重置密钥"按钮
3. 系统会生成新的密钥
4. 确认重置
5. **注意**: 重置后需要重新绑定

#### 删除服务

1. 在服务列表中找到要删除的服务
2. 点击"删除"按钮
3. 确认删除操作

### 5. 授权管理

#### 为单个用户授权服务

1. 进入"服务管理"页面
2. 点击目标服务的"授权管理"按钮
3. 在授权管理页面，点击"添加授权"
4. 选择要授权的用户
5. 点击"确认"

#### 批量授权服务

1. 进入"服务管理"页面
2. 点击目标服务的"授权管理"按钮
3. 在授权管理页面，点击"批量授权"
4. 选择多个用户
5. 点击"确认"

#### 撤销授权

1. 进入"服务管理"页面
2. 点击目标服务的"授权管理"按钮
3. 在已授权用户列表中，点击"撤销"按钮
4. 确认撤销操作

### 6. 审计报表

#### 查看操作日志

1. 进入"审计报表"页面
2. 选择"操作日志"标签
3. 设置筛选条件：
   - **时间范围**: 选择日期范围
   - **用户**: 选择特定用户
   - **操作类型**: 选择操作类型
   - **服务**: 选择特定服务
4. 点击"查询"
5. 查看日志详情

#### 导出操作日志

1. 在操作日志页面
2. 设置筛选条件
3. 点击"导出 CSV"按钮
4. 下载 CSV 文件

#### 查看统计报表

1. 进入"审计报表"页面
2. 选择"统计报表"标签
3. 查看以下报表：
   - **服务访问排行**: Top 20 最常访问的服务
   - **用户活跃排行**: Top 20 最活跃的用户
   - **操作类型分布**: 各类操作的统计
   - **每日访问趋势**: 访问量趋势图

### 7. 系统配置

#### 修改系统配置

1. 进入"系统设置"页面
2. 修改配置项：
   - **登录失败最大尝试次数**: 触发 IP 封禁的失败次数
   - **IP 封禁时长**: 封禁时长（分钟）
   - **Token 有效期**: 管理员/用户 Token 有效期
3. 点击"保存"

#### 查看系统状态

1. 进入"系统设置"页面
2. 选择"系统状态"标签
3. 查看以下信息：
   - **服务状态**: 服务端运行状态
   - **数据库状态**: 数据库连接状态
   - **用户统计**: 用户数量统计
   - **服务统计**: 服务数量统计

## 普通用户操作指南

### 1. 登录系统

#### Web 管理后台登录

1. 打开浏览器，访问 `http://your-server:3001`
2. 输入用户名和密码
3. 点击"登录"按钮

#### PC 客户端登录

1. 启动 GuardVault 客户端
2. 首次使用时，配置服务端地址：
   - 系统会自动扫描局域网内的服务端
   - 或手动输入服务端地址（如 `http://192.168.1.100:3001`）
3. 输入用户名和密码
4. 点击"登录"按钮

### 2. 查看服务列表

登录后，系统会显示您有权限访问的服务列表：

- **服务名称**: 服务的显示名称
- **服务分类**: 服务所属分类
- **账号**: 服务账号
- **状态**: 服务在线状态

### 3. 获取 TOTP 验证码

#### 方式一：在服务列表中查看

1. 在服务列表中找到目标服务
2. 点击服务卡片
3. 系统会显示：
   - **验证码**: 6 位或 8 位数字
   - **倒计时**: 剩余有效时间
   - **复制按钮**: 一键复制验证码

#### 方式二：使用快捷键

- `Ctrl + 1`: 复制第一个服务的验证码
- `Ctrl + 2`: 复制第二个服务的验证码
- `Ctrl + 3~9`: 复制对应服务的验证码
- `Esc`: 清空剪贴板

#### 方式三：使用系统托盘（仅 PC 客户端）

1. 在系统托盘中找到 GuardVault 图标
2. 右键点击图标
3. 选择要复制验证码的服务
4. 验证码自动复制到剪贴板

### 4. 复制验证码

1. 点击"复制"按钮
2. 验证码自动复制到剪贴板
3. 粘贴到需要验证码的应用中
4. **注意**: 30 秒后剪贴板会自动清空

### 5. 修改个人密码

1. 点击右上角用户名
2. 选择"修改密码"
3. 输入当前密码和新密码
4. 点击"确认"

**密码要求**:
- 至少 8 个字符
- 包含大小写字母
- 包含数字
- 包含特殊字符

### 6. 开启/关闭个人 2FA

#### 开启个人 2FA

1. 点击右上角用户名
2. 选择"个人设置"
3. 找到"双因素认证"选项
4. 点击"启用 2FA"
5. 系统显示二维码
6. 使用手机验证码应用扫描二维码
7. 输入验证码确认绑定

#### 关闭个人 2FA

1. 点击右上角用户名
2. 选择"个人设置"
3. 找到"双因素认证"选项
4. 点击"禁用 2FA"
5. 确认操作

**注意**: 关闭个人 2FA 后，登录时不再需要输入验证码。

## 客户端使用指南

### 1. 安装客户端

#### Windows

1. 下载 `GuardVault-Setup-x.x.x.exe`
2. 双击运行安装程序
3. 按照提示完成安装
4. 启动客户端

#### macOS

1. 下载 `GuardVault-x.x.x.dmg`
2. 双击打开 DMG 文件
3. 将 GuardVault 拖拽到 Applications 文件夹
4. 启动客户端

#### Linux

1. 下载 `GuardVault-x.x.x.AppImage`
2. 添加执行权限：`chmod +x GuardVault-x.x.x.AppImage`
3. 运行：`./GuardVault-x.x.x.AppImage`

### 2. 配置服务端地址

#### 自动发现

1. 启动客户端
2. 系统自动扫描局域网内的服务端
3. 在列表中选择服务端
4. 点击"连接"

#### 手动配置

1. 在配置页面点击"手动配置"
2. 输入服务端地址（如 `http://192.168.1.100:3001`）
3. 点击"测试连接"
4. 连接成功后点击"保存"

### 3. 系统托盘功能

#### 右键菜单

- **显示主窗口**: 打开主窗口
- **复制验证码**: 快速复制常用服务的验证码
- **重新连接**: 重新连接服务端
- **退出**: 退出客户端

#### 左键点击

- 显示主窗口
- 或显示验证码列表（根据配置）

### 4. 开机自启

1. 点击客户端右上角设置图标
2. 选择"开机自启"
3. 启用该选项

**注意**:
- Windows: 添加到启动文件夹
- macOS: 添加到登录项
- Linux: 添加到 `~/.config/autostart`

### 5. 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + 1~9` | 复制对应服务的验证码 |
| `Esc` | 清空剪贴板 |
| `Ctrl + Q` | 退出客户端 |
| `Ctrl + ,` | 打开设置 |

### 6. 通知设置

1. 点击客户端右上角设置图标
2. 选择"通知设置"
3. 配置通知选项：
   - **验证码即将过期**: 提前 5 秒通知
   - **服务端离线**: 服务端断开连接时通知
   - **验证码复制成功**: 复制成功时通知

**管理员操作:**

1. 创建用户账号
2. 分配到对应部门
3. 授权相关服务

2. 登录系统

4. 重新授权

## 安全注意事项

### 1. 密码安全

- 使用强密码（至少 8 位，包含大小写字母、数字、特殊字符）
- 定期更换密码
- 不要与他人共享密码
- 不要在多个系统使用相同密码

### 2. TOTP 密钥安全

- 不要截图分享密钥二维码
- 不要将密钥明文存储在不安全的地方
- 密钥泄露后立即重置
- 定期检查服务密钥使用情况

### 3. 客户端安全

- 从官方渠道下载客户端
- 及时更新客户端版本
- 不要在公共电脑上使用
- 使用后及时锁定屏幕

### 4. 网络安全

- 使用 HTTPS 访问（生产环境）
- 不要在公共 WiFi 下使用
- 注意钓鱼网站
- 定期检查访问日志

### 5. 审计与监控

- 定期查看操作日志
- 关注异常访问行为
- 及时报告安全问题
- 配合安全审计

### 6. 数据备份

- 定期备份数据库
- 备份配置文件
- 测试恢复流程
- 保存备份到安全位置

## 常见问题

### 1. 忘记密码怎么办？

联系管理员重置密码。

### 2. 验证码不正确怎么办？

- 检查系统时间是否准确
- 确认验证码未过期
- 尝试重新获取验证码
- 如仍有问题，联系管理员

### 3. 客户端无法连接服务端？

- 检查网络连接
- 确认服务端地址正确
- 检查防火墙设置
- 联系管理员确认服务端状态

### 4. 如何导出我的服务列表？

目前不支持用户导出，如需导出请联系管理员。

### 5. 可以在多个设备上使用吗？

可以，但建议在主要设备上使用，避免密钥泄露风险。

### 6. 如何更换手机？

1. 在新手机上安装验证码应用
2. 重新扫描二维码绑定
3. 删除旧手机上的记录

### 7. 服务端离线怎么办？

- 检查网络连接
- 联系管理员确认服务端状态
- 等待服务端恢复
